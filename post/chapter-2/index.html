Yelp Recommender : The Places You Didn't Know You Wanted to Visit

October 15, 2022

Yelp is an online platform where users can rate and write reviews on local businesses. Users can also browse the website to search for local businesses of their interest.

To facilitate users’ browsing on Yelp’s website, this project aims to make customized recommendations on local businesses to every user.

More than 200 thousand consumer reviews in the City of Santa Barbara, CA on Yelp were analyzed to study the attributes of users and businsesses that lead to good customer rating, and to implement various techniques for the machine to learn a given user’s preferences.

Based on the findings from the above, a program was written to compute two things given a user ID:

  1. Find the user’s favourite business (i.e., most highly-reviewed business), and then sort all the other businesses in the order of similarity to the favourite

  2. Predict the user’s likely ratings on all businesses on a continuous scale between 0 and 5.

Finally, a hybrid recommender was created such that it outputs ten businesses that are similar to the given user’s favourite and/or predicted to be enjoyed by the given user.

This project was the capstone project for my BrainStation Data Science Bootcamp.

Why Did I Do This Project?

  • To study the characteristics of users and businesses leading to good experience and eventually good reviews.

  • To improve user experience with the service by reducing browing time and effort and leave the impression that the service provider (Yelp) knows exactly what the users want - even when the users don’t know themselves!

  • To better understand how Data Science can be utilized to improve customer satisfaction in the commercial setting.

Data Cleaning

As the first step, the original data in json format was converted to a more workable format (i.e. csv). Then, features were added and modified to be suitable for exploratory data analysis and modelling.

Multiple tables - i.e. business, user, review, check-ins - were cleaned, reformatted and combined into a large table. Each row of this table contained the user ID, business ID, user attributes, business attributes, user review, and rating.

Some critical information - i.e. the business hours, categories, and attributes - were formatted as texts containing many typos and exceptions, which called for extensive cleaning and normalization. This is done by using Regular Expression (RegEx) substitutions and user-defined element-wise functions.

Furthermore, a discrete 1-to-5 star rating system is difficult to work with because one cannot rank businesses that are rated the same stars. To make the scale continuous, the users’ reviews were used to compute the sentiment and was combined with their star ratings to create a continuous rating from 0 to 5. This adjusted rating is what I aimed to predict for my regression models.

In General, What Causes High and Low Ratings?

Based on the exploratory data analysis, in general, it was found that there was no definitive attribute of a business or user that was highly correlated with the user rating. This signifies that every user has their own preferences, and the preferences are somewhat evenly distributed such that no attribute is related to a high or low rating.

The only attributes relatively highly correlated to user ratings were the Yelp business rating, and user’s average stars (rating).

How Should We Make Recommendations?

So what is a good recommender? One that recommends similar businesses to the ones a user have already tried? One that recommends businesses that are generally liked by most other users? Given the cleaned data and some insights on the business-user relationship, a couple different options for recommendation algorithms were considered:

  1. Content-Based Recommendation :

    • Create a matrix containing all pairwise cosine similarities between businesses

    • Given a user’s favourite (i.e., most highly rated) business, sort all the other businesses not yet reviewed by the user in the descending order of cosine similarity.

    • From the sorted list of businesses, make n recommendations.

    • Disadvantages : What if the user wanted to be recommended something completely different from what they have already tried? What if even the user’s most highly rated business is rated low (e.g. 0-2)?

    T-SNE Plot for an Arbitrary User. Note the Recommendations based on Content-Based Filtering are Close to the User’s Favourite

  2. Collaborative Filtering :

    • Two types considered : user-based KNN approach, and SVD matrix factorization approach

    • User Based : Given a user U, find all other users who also rated the same businesses as the user U. Rank these other users in the descending order of cosine similarity to the user U in terms of their ratings on such businesses. Then, for every business not rated by U, determine the average rating of the top n users most similar to U as the predicted rating of U on each of those businesses.

    • SVD Matrix Factorization :

      • An m x n user-item review matrix is factored to the multiplication of two matrices - an m x p user matrix, and p x n item matrix.

      • In our context, the p represents some arbitrarily created businsess features.

      • In the m x p user matrix, the “p-dimension” is the users’ preferences for the p different features (e.g., how much a given user prefers a classy atmosphere). In the p x n item matrix, the “p-dimension” is how much a business belongs to the features (e.g., how classy an atmosphere a given restaurant has).

      • In other words, the matrix factorization method algorithm attempts to quantify each user’s preferences and businesses’ attributes given the input data containing user-item-rating.

      • By matrix multiplication of these factored matrices, predict the pairwise ratings between all users and businesses.

      • Finally, make n recommendations in the descending order of predicted ratings.

    • Disadvantage : Not as effective when there are not many users who rated the same business.

    SVD Factorization. Image referenced from Real Python

  3. Regression (with Neural Network) :

    • Using the review data, train a deep neural network to learn the relationship between all the user/business attributes and user ratings.

    • Then, given a user id, predict all pairwise ratings between the given user and all other businesses.

    • Finally, make n recommendations in the descending order of predicted ratings.

    • Disadvantage : The regression model will pay an especially closer attention to the features highly correlated with the label - i.e., the business rating and user average stars. Therefore, rather than focusing on each user’s unique preferences, the model might lean toward recommending popular businesses.

    T-SNE Plot for an Arbitrary User. Note the Recommendations based on NN Regression are Not Close to the User’s Favourite

Which Recommender to Choose? Can We Mix Them Up?

Each recommender has their pros and cons. That said, content-based filtering was seen to be very effective in selecting similar businesses to one’s favourite, while neural network regressor was good at making accurate predictions on user ratings regardless of their preferences.

Combining these two models, our final, ‘hybrid’ recommender is created. This one recommends businesses that are: A) similar to a given user’s favourite and/or B) predicted to be liked by the user regardless of the similarity to any of the user’ previously tried businesses.

But we can’t always recommend similar businesses to the users’ most highly rated one; what if the user hated all the businesses they have tried and rated all of them low on Yelp? In this case, we can’t recommend businesses that are similar to what that user hated.

As such, we mainly consider two cases:

  • Case 1 : When a user was satisfied with at least one of businesses they have tried.
    • When at least one business is rated high (i.e. >= 4) by a given user, we make 10 recommendations in the following way:
      1. Get the full list of businesses not yet reviewed by the user, in the descending order of cosine similarity to the user’s most highly rated business (content-based filtering)
      2. Get the full list of businesses not yet reviewed by the user, in the descending order of ratings predicted using neural network
      3. From the two lists each created in step 1. and 2., select the first five businesses appearing in both lists.
      4. From the content-based recommendations, select the top three businesses not appearing in the businesses selected in 3.
      5. From the neural network recommendations, select the top two businesses appearing in neither list from 3. and 4.

T-SNE Plot for an Arbitrary User Corresponding to Case 1. Note User Favourite vs Recommendations

  • Case 2 : When a user was not satisfied with any of the businesses they have tried.
    • Where no business is rated high (i.e. < 4) by a given user, we make 10 recommendations in the following way:
      1. Get the full list of businesses not yet reviewed by the user, in the descending order of ratings predicted using neural network
      2. From the list created in step 1., select ten businesses for recommendation.

T-SNE Plot for an Arbitrary User Corresponding to Case 2. Note User Favourite vs Recommendations

How Accurate is Our Model?

Different metrics were used to evaluate each of the recommendation algorithms:

  • Content-based filtering was evaluated based on the mean cosine distance of the recommended businesses from the given user’s favourite (or top n favourites)
  • Regressor was evaluated on both Root Mean Square Error (RMSE) and Mean Absolute Error (MAE).

The error metric(s) of each algorithm is summarized as below:

Algorithm Error Type(s) & Value(s)
Content Based Mean Cosine Distance : 0.2
NN Regressor RMSE : 0.96 , MAE : 0.68
